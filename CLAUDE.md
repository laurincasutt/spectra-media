This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev      # Start dev server on http://localhost:3035
npm run build    # Production build (also runs next-sitemap via postbuild)
npm run start    # Start production server on port 3035
npm run lint     # ESLint via next lint
```

There are no tests in this project.

## Architecture

### Content layer (single source of truth)
All German copy lives in **`data/content.ts`** — the only file that needs editing for text, stats, links, or case study data. It exports typed objects: `nav`, `home`, `caseStudies` (array), `service`, `prozess`, `gruender`. The `CaseStudy` interface is also defined here.

To add a new case study: add an entry to the `caseStudies` array with a unique `slug`; `generateStaticParams` in `app/case-studies/[slug]/page.tsx` picks it up automatically.

### Server / Client component split
Every animated page uses a two-file pattern to allow both `metadata` exports and Framer Motion:
- `page.tsx` — Server Component, exports `metadata`, renders a `*Client.tsx`
- `*Client.tsx` — `"use client"`, contains all Framer Motion markup

Example: `app/service/page.tsx` → `app/service/ServiceClient.tsx`. The same pattern is used for `gruender`, `case-studies` index, and `case-studies/[slug]`.

### Animations
Shared Framer Motion variants are in **`lib/animations.ts`**. Use `staggerContainer` as a parent with `variants` prop, `fadeUpVariants` / `slideInLeft` / `scaleIn` as children. Always pass `viewport={viewportOnce}` (defined in animations.ts) to `whileInView` calls so animations fire once.

### Styling (Tailwind CSS v4)
This project uses **Tailwind v4**, which has no `tailwind.config.ts`. Brand tokens are defined in `app/globals.css` under the `@theme` directive as CSS custom properties (e.g. `--color-brand`, `--color-surface-card`). Custom utilities (`gradient-text`, `bg-grid`, `glow-blue`) are also defined there as plain CSS classes.

In practice, most components use raw hex values inline (e.g. `bg-[#09090B]`) rather than the token names, for consistency with design.

### SEO
- Title template `"%s | Spectra Media GmbH"` is set in `app/layout.tsx`; each page sets its own `title` string.
- Schema.org JSON-LD is inlined via `<script dangerouslySetInnerHTML>` in page Server Components (`/` has Organization + FAQPage schemas, case study detail pages have Article schema).
- `sitemap.xml` and `robots.txt` are auto-generated by `next-sitemap` on every `npm run build` (postbuild hook). Config is in `next-sitemap.config.js`. `/impressum` and `/datenschutz` are excluded from the sitemap and set to `noindex`.

### Environment variables
Copy `.env.example` to `.env.local`. `SITE_URL` is used by `next-sitemap` and `metadataBase`. `NEXT_PUBLIC_WHATSAPP_NUMBER` is read at runtime in `Navbar.tsx` (falls back to the value hardcoded in `data/content.ts`).

### Deployment
`vercel.json` sets `framework: nextjs` and `regions: ["fra1"]`. Push to `main` to deploy. No special build command needed — Vercel runs `npm run build` which includes the sitemap postbuild step.
